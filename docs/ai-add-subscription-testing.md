## 测试步骤

### 测试1: 基本添加流程

**目标**: 验证AI能够识别并添加新订阅

**步骤**:
1. 打开浏览器访问 http://localhost:8501
2. 选择用户（如果需要）
3. 进入 "AI助手" 页面
4. 在对话框输入：`帮我添加Netflix订阅，每月79元`
5. 按回车或点击发送

**预期结果**:
- AI返回友好的确认消息
- 页面显示 "✅ AI已识别到订阅信息"
- 显示订阅详情卡片：
  - 服务名称: Netflix
  - 价格: CNY 79
  - 计费周期: 每月
  - 分类: 娱乐
- 显示三个按钮：✅ 确认添加、✏️ 修改信息、❌ 取消

**操作**:
- 点击 "✅ 确认添加"

**验证**:
- 显示成功消息：`✅ 已成功添加 Netflix 订阅！`
- 页面刷新，确认卡片消失
- 进入 "订阅管理" 或 "Dashboard" 查看，确认Netflix已添加到订阅列表

## 已知限制

1. **单个订阅识别**: 当前版本一次只能识别一个订阅
   - 不支持: `同时添加Netflix和Disney+`
   - 未来可增强

2. **依赖AI模型**: 识别准确度取决于Gemini API
   - 如果AI不可用，功能降级
   - 仍可手动通过 "添加订阅" 页面添加

3. **简单重复检测**: 仅基于名称的模糊匹配
   - 不检测价格或周期差异
   - 可能将同一服务的不同套餐标记为重复

## 故障排查

### 问题: AI没有识别订阅意图

**可能原因**:
- Gemini API不可用
- 用户表达方式太复杂

**解决方案**:
- 检查AI助手状态（页面顶部）
- 使用更直接的表达："添加XX订阅，每月XX元"
- 降级方案：使用手动添加订阅功能

### 问题: 确认卡片不显示

**可能原因**:
- Session state未正确设置
- 页面未刷新

**解决方案**:
- 刷新页面重试
- 检查浏览器控制台错误
- 查看Streamlit终端日志

### 问题: 添加失败

**可能原因**:
- 数据库连接问题
- 必填字段缺失

**解决方案**:
- 检查 `data/subscriptions.json` 文件权限
- 使用修改功能补充缺失字段
- 查看错误消息详情

## 后续优化建议

1. **批量添加**: 支持一次添加多个订阅
2. **智能更新**: 识别更新订阅的意图（"把Netflix改成89元"）
3. **删除订阅**: 支持通过对话删除订阅
4. **历史记录**: 记录AI添加的订阅，支持撤销
5. **更智能的分类**: 基于更多上下文推断分类
6. **提醒设置**: 添加时自动设置续费提醒
